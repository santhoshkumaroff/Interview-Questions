Q03 What is a decorator in Python?
Definition
A decorator is a function that wraps another function to extend or modify its behavior without changing the original code.
Simple Example
Python
def log_call(func):
    def wrapper(*args):
        print(f"Calling {func.__name__}")
        return func(*args)
    return wrapper

@log_call
def greet(name):
    print(f"Hello {name}")

greet("Alice")
# Output: Calling greet → Hello Alice
Industry Example

Flask uses @app.route() as a decorator to register URL endpoints.

Follow-up Questions

Q04 What is the difference between deep copy and shallow copy?
Definition
Shallow copy copies the outer object but keeps references to inner objects. Deep copy copies everything recursively — a fully independent clone.
Simple Example
Python
import copy
a = [[1, 2], [3, 4]]
b = copy.copy(a)       # Shallow
c = copy.deepcopy(a)   # Deep
a[0][0] = 99
print(b[0][0])  # 99 (shared inner!)
print(c[0][0])  # 1  (independent)

Q17 What is Overfitting and Underfitting?
Definition
Overfitting: model memorizes training data — poor on new data (high variance). Underfitting: model is too simple — poor on training and test data (high bias).
Industry Example

A fraud detection model that memorizes past frauds but fails on new fraud patterns = overfitted. A model that says everything is "not fraud" = underfitted.

Solutions

Overfitting → Regularization (L1/L2), Dropout, more data, pruning. Underfitting → More features, complex model, reduce regularization.

Follow-up Questions

Q18 What is the Bias-Variance Tradeoff?
Definition
Bias = error from oversimplified model. Variance = error from model being too sensitive to training data. You must balance them — reducing one usually increases the other.
Formula

Total Error = Bias² + Variance + Irreducible Noise

Follow-up Questions

T05Explain Linear Regression in detail.
Definition
Linear Regression models the relationship between a dependent variable y and one or more independent variables x by fitting a straight line: y = mx + b.

T07What is Random Forest and how does it work?
Definition
Random Forest is an ensemble of decision trees trained on random subsets of data (bagging) and random feature subsets. Final prediction is by majority vote (classification) or average (regression).

Individual trees overfit. Averaging many diverse trees (built on different data samples) reduces variance — ensemble wisdom overcomes individual model noise.

T08Explain K-Means Clustering.
Definition
K-Means partitions n data points into k clusters by iteratively assigning each point to the nearest centroid and updating centroids until convergence.
Algorithm Steps

1. Initialize k centroids randomly. 2. Assign each point to nearest centroid. 3. Recompute centroid = mean of assigned points. 4. Repeat steps 2-3 until centroids don't move.

Elbow Method

Plot inertia (sum of distances to centroid) vs k. The "elbow" point where the curve bends is the optimal k.

def two_sum(nums, target):
    seen = {}
    for i, n in enumerate(nums):
        complement = target - n
        if complement in seen:
            return [seen[complement], i]
        seen[n] = i
    return []

def merge_sorted(a, b):

    i = j = 0
    while i < len(a) and j < len(b):
        if a[i] < b[j]:
            result.append(a[i]); i += 1
        else:

C18 Linear Regression from scratch using NumPy
Python
import numpy as np

class LinearRegression:
    def __init__(self, lr=0.01, epochs=1000):
        self.lr = lr; self.epochs = epochs
        self.w = self.b = 0

    def fit(self, X, y):
        n = len(X)
        for _ in range(self.epochs):
            y_pred = self.w * X + self.b
            dw = (-2/n) * np.sum(X * (y - y_pred))
            db = (-2/n) * np.sum(y - y_pred)
            self.w -= self.lr * dw
            self.b -= self.lr * db

    def predict(self, X):
        return self.w * X + self.b

from collections import Counter
import numpy as np

def euclidean(a, b):
    return np.sqrt(np.sum((np.array(a)-np.array(b))**2))

def knn_predict(train_X, train_y, test_point, k=3):
    distances = [(euclidean(x, test_point), y)
                 for x, y in zip(train_X, train_y)]
    distances.sort(key=lambda x: x[0])
    top_k = [y for _, y in distances[:k]]
    return Counter(top_k).most_common(1)[0][0]

What is RAG and can you describe a project where you used it?
ME
Candidate
RAG stands for Retrieval-Augmented Generation. It solves the limitation of LLMs having outdated or no knowledge of specific documents. In RAG, we first index documents as embeddings in a vector database. The LLM then generates an answer grounded in those retrieved facts rather than hallucinating. What is the difference between precision and recall? When would you optimize for each?

Candidate
Precision measures: of all cases we predicted as positive, how many were actually positive? Recall measures: of all actual positive cases, how many did we correctly catch? There's a tradeoff. For cancer screening, we'd optimize for high Recall — we'd rather have false alarms (false positives) than miss actual cancer cases. For spam detection, we'd optimize for high Precision — we'd rather let some spam through than accidentally block important emails.
